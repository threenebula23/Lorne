# Архитектура интерфейса TCA (TUI OVERVIEW)

Интерфейс TCA построен на фреймворке **Textual** (Python). Это современная библиотека для создания терминальных приложений с поддержкой мыши, слоев, CSS и асинхронности.

## Для всех: Как это устроено?
Представьте, что TCA — это конструктор LEGO. 
- Есть **главное окно** (`TCAApp`), которое держит всё вместе.
- Внутри окна есть **панели** (редактор кода, дерево файлов, чат, терминал).
- Между панелями есть **"мост"** (`TUIBridge`) — это специальный канал связи, по которому агент (искусственный интеллект) отправляет сообщения в интерфейс (например, "выведи текст в чат" или "покрась эту кнопку в зеленый").

---

## Для разработчиков: Основные компоненты

### 1. Главное приложение (`Interface/tui_app.py`)
Класс `TCAApp` — это входная точка.
- **Compose (Разметка)**: Метод `compose()` определяет, где какая панель находится. Мы используем `Horizontal` и `Vertical` контейнеры для сетки.
- **ResizeHandle**: Специальный виджет для перетаскивания границ панелей мышкой.
- **Bindings (Горячие клавиши)**: Здесь прописаны все `Ctrl+Q`, `Ctrl+S` и прочее.

### 2. Мост (`Interface/tui_bridge.py`) — КРИТИЧЕСКИ ВАЖНО
Поскольку агент работает в отдельном потоке (Thread), он не может напрямую обращаться к элементам интерфейса (это вызовет ошибку).
- **Thread Safety**: Все вызовы в мосте обернуты в `self.app.call_from_thread(...)`.
- **Методы**:
  - `on_thought()`: Отправляет мысли агента в `AIChatPanel`.
  - `request_confirmation()`: Ставит поток агента на паузу и показывает пользователю окно (Screen) с кнопками "Да/Нет". Возвращает результат агенту после того, как пользователь нажмет кнопку.

### 3. Панели (`Interface/panels/`)
Каждая панель — это отдельный класс, наследуемый от `Static` или `Vertical`.
- **`ai_chat.py`**: Управляет историей сообщений, выбором моделей и кнопкой "Стоп". Использует `RichLog` для красивого вывода Markdown.
- **`code_editor.py`**: Обёртка над `TextArea`, дополненная поддержкой вкладок (Tabs) и интеграцией с Jupyter Notebook.
- **`file_explorer.py`**: Дерево файлов (`Tree`), которое умеет обновляться при изменении файлов на диске.

## Поток данных (Data Flow)
1. Пользователь нажимает **Enter** в чате.
2. `AIChatPanel` отправляет событие `ChatSubmitted`.
3. `TCAApp` ловит событие и вызывает функцию `on_chat_submit` в `Agent/agent.py`.
4. Агент начинает думать. В процессе он вызывает `bridge.on_thought("...")`.
5. Мост перебрасывает сообщение обратно в `AIChatPanel`, и вы видите текст на экране.

## Стилизация
Все цвета и размеры задаются в файле `Interface/tui_app.tcss`. Это аналог CSS для веба. Вы можете менять отступы, границы и фоны, не трогая логику на Python.
