# Документация инструмента: versioning_tool.py (Снимки и Откат)

Этот инструмент обеспечивает безопасность правок агента, позволяя мгновенно восстановить предыдущее состояние любого файла.

## Для всех: Что это делает?
Это "кнопка отмены" (Undo) для файлов, но работающая на уровне всей файловой системы.
- Перед тем как агент меняет файл (через `edit_file` или `write_file`), система автоматически делает "снимок" его содержимого.
- **`list_file_versions`**: Агент может посмотреть список всех сохраненных версий конкретного файла.
- **`rollback_file`**: Агент (или вы) можете вернуть файл к любой из предыдущих версий, если что-то пошло не так.

---

## Для разработчиков: Разбор кода (подробно)

### Зависимости (Строки 9–18)
- Инструмент является тонкой оберткой над внутренним модулем `Agent.versioning`. Сам модуль `versioning` обычно использует SQLite для хранения истории изменений.

### Инструменты

#### `list_file_versions` (Строки 22–25)
- Возвращает список объектов версий: ID версии, время сохранения, размер и краткую заметку (например, "до команды edit_file").

#### `rollback_file` (Строки 29–34)
- **Умный откат**: 
  - Если передать `version_id`, файл восстановится именно к этой точке.
  - Если оставить `version_id` пустым, восстановится **последняя** сохраненная версия. Это самый частый сценарий использования, когда агент понимает: "Ой, я сломал код, надо всё вернуть назад".

---

## Особенности
1. **SQLite база**: Все версии хранятся в скрытом файле `.tca_versions.sqlite` в корне проекта. Это экономит место по сравнению с хранением копий файлов.
2. **Автоматизм**: Агент никогда не вызывает сохранение версии вручную — это происходит автоматически "под капотом" других инструментов. Инструменты отката нужны ему только для исправления ошибок.

---

## Примеры использования

**Посмотреть список версий:**
```python
list_file_versions(path="main.py")
```

**Откатиться к последней версии:**
```python
rollback_file(path="main.py")
```
