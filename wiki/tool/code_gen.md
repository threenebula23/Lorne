# Документация инструмента: code_gen.py (Умная генерация кода)

> **Для вызова у модели:** **`code_file_tool`** (`create` \| `append`) — обёртка над `create_code_file` / `append_code_snippet`. См. [COMPACT_TOOLS.md](../COMPACT_TOOLS.md).

Этот инструмент помогает агенту создавать новые файлы с программным кодом, автоматически подбирая правильное расширение файла и исправляя ошибки форматирования.

## Для всех: Что это делает?
Это помощник по созданию файлов. Если агент хочет создать программу на Python, но забыл добавить `.py` в названии файла, этот инструмент исправит это за него.
- **`create_code_file`**: Создать новый файл с кодом. Вы говорите "сделай калькулятор на языке Python", и инструмент создаст `calculator.py`.
- **`append_code_snippet`**: Добавить кусочек кода в конец уже существующего файла. Полезно для добавления новых функций или импортов.

## Для разработчиков: Разбор кода (подробно)

### Маппинг расширений (Строки 20–64)
- Словарь `_LANG_TO_EXT` сопоставляет названия языков (от "python" до "dockerfile") с их стандартными расширениями. Это избавляет агента от необходимости помнить все расширения.

### Логика именования (Строки 71–90)
- **`_apply_extension`**: Главная функция-корректор.
  - Если у файла нет расширения — добавляет его на основе языка.
  - Если файл ошибочно назван `.txt`, но внутри код — меняет `.txt` на правильное расширение (например, `.js`).
  - Учитывает исключения (например, `Dockerfile` всегда без расширения).

### Чистка текста (Строки 93–102)
- **`_normalize_code_text`**: Иногда LLM передает переводы строк как символы `\n` (двойное экранирование). Функция "разворачивает" такие строки, превращая их в нормальный многострочный текст.

### Основные функции

#### `create_code_file` (Строки 106–133)
- По сути, это обертка над обычной записью файла, но с проверкой расширений и созданием бэкапа (snapshot) перед записью.

#### `append_code_snippet` (Строки 137–168)
- Открывает файл, проверяет, заканчивается ли он пустой строкой (если нет — добавляет её для красоты), и приклеивает новый код в самый конец.

---

## Особенности
1. **Безопасность**: Перед перезаписью файла всегда вызывается `save_version`, так что вы можете откатить изменения, если агент что-то сломал.
2. **Удобство**: Инструмент сам создает папки, если вы указали путь к файлу в еще не существующей директории.

---

## Примеры использования

**Создание файла без указания расширения:**
```python
create_code_file(filepath="utils", language="python", code="def hello(): print('hi')")
# Результат: будет создан файл utils.py
```

**Добавление импорта в конец:**
```python
append_code_snippet(filepath="main.py", snippet="import os", language="python")
```
