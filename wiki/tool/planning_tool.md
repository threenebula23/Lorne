# Документация инструмента: planning_tool.py (Планирование задач)

> **Для вызова у модели:** один тул **`plan_tool`** (`action`: `save` \| `load` \| `update` \| `clear`) — см. [COMPACT_TOOLS.md](../COMPACT_TOOLS.md). Ниже описаны функции модуля `planning_tool.py`, на которые `plan_tool` делегирует.

Этот инструмент позволяет агенту разбивать сложные задачи на последовательность мелких шагов, отслеживать прогресс и не терять фокус при длительной работе.

## Для всех: Что это делает?
Это "список дел" или "чек-лист" агента.
- **`save_plan`**: Агент записывает план действий (например: 1. Прочитать код, 2. Исправить баг, 3. Запустить тесты).
- **`load_plan`**: Агент проверяет, что уже сделано, а что осталось.
- **`update_plan`**: Агент отмечает шаги как выполненные или переводит их в статус "в процессе".
- **`clear_plan`**: Удалить старый план, когда задача полностью завершена.

Вы можете видеть этот план в интерфейсе TCA, что позволяет вам контролировать, чем сейчас занят агент.

---

## Для разработчиков: Разбор кода (подробно)

### Хранение данных (Строки 15–32)
- План сохраняется в JSON-файл по пути `.tca/plan.json` в корне вашего проекта. Это делает план "живучим" — он не теряется при перезагрузке программы или падении агента.

### Инструменты

#### `save_plan` (Строки 35–66)
- Принимает заголовок и список строк-шагов.
- **Умное сохранение (Строки 40–56)**: Если файл плана уже существовал, инструмент старается сопоставить новые шаги со старыми по тексту. Если шаг уже был помечен как "выполнен", его статус сохранится. Это позволяет агенту обновлять архитектуру плана, не теряя информацию о прогрессе.

#### `update_plan` (Строки 76–101)
- Позволяет менять статус шага по его индексу.
- **Статусы**: `pending` (ожидает), `in_progress` (в работе), `completed` (завершено), `blocked` (блокировано).
- Поддерживает поле `note` для кратких комментариев к шагу.

#### `clear_plan` (Строки 104–117)
- Удаляет файл плана. **Важно**: требует подтверждения пользователя через терминал (`input`), чтобы агент случайно не стер важный чек-лист.

---

## Особенности
1. **Автоматизация**: Агент часто вызывает `save_plan` в самом начале диалога, чтобы структурировать свою работу.
2. **Маршрутизация**: В TUI-интерфейсе есть отдельная вкладка для отображения этого плана в реальном времени.

---

## Примеры использования

**Создание плана:**
```python
save_plan(
    title="Рефакторинг API", 
    steps=["Создать новые роуты", "Перенести логику", "Удалить старый код"]
)
```

**Отметка о выполнении:**
```python
update_plan(step_index=0, status="completed", note="Роуты созданы в файле api_v2.py")
```
