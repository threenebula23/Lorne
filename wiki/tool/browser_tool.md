# Документация инструмента: browser_tool.py (Автоматизация браузера)

> **Для вызова у модели (режим Agent):** Node Playwright обёрнут в **`headless_browser`**; опционально добавляется **`playwright_sync`** (Python), если в Settings включён Python Playwright. Соответствие `action` ↔ старые имена — в [COMPACT_TOOLS.md](../COMPACT_TOOLS.md). Ниже — реализация в `browser_tool.py` / `playwright_sync_tool.py`.

Этот инструмент позволяет агенту не просто искать текст, но и "видеть" сайты почти так же, как человек — загружать их с выполнением JavaScript, кликать по кнопкам и делать скриншоты.

## Для всех: Что это делает?
Это продвинутый способ работы с сайтами, которые сложно прочитать обычным поиском (например, современные приложения на React/Vue, где контент загружается динамически).
- **`browser_get_text`**: Зайти на сайт, подождать загрузки и вытащить текст из нужной части страницы (по селектору).
- **`browser_screenshot`**: Сделать снимок экрана сайта и сохранить его в файл.
- **`browser_click_and_get`**: Нажать кнопку или ссылку на сайте, подождать и прочитать результат.
- **`browser_evaluate`**: Выполнить произвольный JavaScript код прямо в браузере (например, пролистать страницу вниз).

## Для разработчиков: Разбор кода (подробно)

### Как это устроено? (Строки 17–45)
Инструмент не держит постоянно открытый браузер в памяти. Он работает через **Playwright CLI**.
- **`_run_playwright_script`**: Эта функция принимает строку с кодом на JavaScript (Node.js + Playwright), создает временный файл `.mjs`, запускает его через системный `node` и возвращает результат (stdout/stderr). После выполнения временный файл удаляется.

### `browser_get_text` (Строки 49–83)
- Использует безголовый (headless) запуск Chromium.
- Переходит по URL и ждет события `domcontentloaded`.
- Ищет элемент по CSS-селектору (по умолчанию `body`).
- Возвращает текстовое содержимое элемента, ограниченное 5000 символами.

### `browser_screenshot` (Строки 87–122)
- Делает снимок страницы.
- Параметр `full_page=True` позволяет снять всю ленту сайта целиком, а не только то, что видно на экране.
- Сохраняет файл по указанному пути.

### `browser_click_and_get` (Строки 126–165)
- Логика: Навигация → Клик по `click_selector` → Пауза (по умолчанию 3 секунды для загрузки результата) → Чтение из `result_selector`.
- Полезно для раскрытия выпадающих списков или перехода по меню в SPA-приложениях.

## Особенности
1. **Требуется Node.js**: Если на компьютере не установлен Node.js и Playwright, инструмент работать не будет.
2. **Headless**: Браузер запускается в фоновом режиме, вы не увидите открывающихся окон на экране.
3. **Безопасность**: Код запускается в изолированном процессе Node.js с ограничением времени выполнения (`timeout`).

## Примеры использования

**Сделать скриншот главной страницы проекта:**
```python
browser_screenshot(url="http://localhost:3000", output_path="ui_check.png")
```

**Нажать на кнопку "Детали" и прочитать текст:**
```python
browser_click_and_get(
    url="https://example.com", 
    click_selector="button#details",
    result_selector=".info-box"
)
```
