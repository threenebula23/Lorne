# Документация инструмента: interactive.py (Общение с пользователем)

> **Для вызова у модели:** **`ask_user`** остаётся отдельным тулом. См. [COMPACT_TOOLS.md](../COMPACT_TOOLS.md).

Этот инструмент позволяет агенту задавать вам вопросы прямо во время выполнения задачи.

## Для всех: Что это делает?
Это способ агента "посоветоваться" с вами.
- **`ask_user`**: Агент выводит вопрос на экран и ждет, пока вы напечатаете ответ.
- Используется, когда:
  - Нужно выбрать один из нескольких вариантов решения.
  - Нужно уточнить детали задания.
  - Агенту нужно подтверждение на совершение действия, которое он считает рискованным.

Ваш ответ возвращается агенту как текст, и он продолжает работу, учитывая ваши пожелания.

---

## Для разработчиков: Разбор кода (подробно)

### Логика работы (Строки 8–33)
Инструмент умеет подстраиваться под режим запуска (TUI или консоль).

1. **Режим IDE (TUI) (Строки 10–15)**:
   - Пытается получить доступ к `TUIBridge`.
   - Если мост активен, вызывает `bridge.request_input(question)`. В интерфейсе TCA появится всплывающее окно (диалог) для ввода текста.

2. **Консольный режим (Строки 17–33)**:
   - Если это обычный терминал, используется стандартная функция Python `input()`.
   - Если ввод недоступен (например, программа запущена в фоне без терминала), инструмент вернет ошибку `Terminal input unavailable`, чтобы агент не "завис" в бесконечном ожидании.

---

## Особенности
1. **Блокировка**: Выполнение программы агента **полностью приостанавливается**, пока вы не дадите ответ.
2. **Безопасность**: Это единственный инструмент, который позволяет агенту получать свободный текстовый ввод от пользователя в середине рабочего процесса.

---

## Примеры использования

**Уточнение задачи:**
```python
ask_user("Я вижу два файла настроек: config.json и settings.yml. Какой из них нужно использовать?")
```

**Запрос на выбор действия:**
```python
ask_user("Тесты не прошли. Попробовать исправить код или просто показать лог ошибок?")
```
