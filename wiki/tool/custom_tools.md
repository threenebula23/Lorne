# Документация инструмента: custom_tools.py (Пользовательские расширения)

Этот инструмент позволяет вам самостоятельно расширять возможности агента TCA, обучая его новым командам без изменения основного кода программы.

## Для всех: Что это делает?
Это "слот для модулей расширения". 
1. Вы пишете небольшой скрипт на Python с нужной вам функцией.
2. Сохраняете его в специальную папку `~/.tca_custom_tools/`.
3. При следующем запуске агент TCA "увидит" этот инструмент и сможет им пользоваться так же, как и встроенными.

Например, вы можете создать инструмент "отправить сообщение в Slack" или "проверить погоду", и агент сможет вплетать эти действия в решение ваших задач.

---

## Для разработчиков: Разбор кода (подробно)

### Загрузчик (Строки 43–82)
- **`load_custom_tools`**: Сканирует папку `~/.tca_custom_tools/` на наличие `.py` файлов.
- Использует модуль `importlib.util` для динамической загрузки файлов как Python-модулей в рантайме.
- Ищет в загруженных модулях объекты, которые являются экземплярами `BaseTool` (через декоратор `@tool` из LangChain).

### Управление файлами (Строки 117–193)
- **`add_custom_tool`**: Позволяет агенту самому создавать новые инструменты. Он записывает код в файл, проверяет его на синтаксические ошибки (пытаясь загрузить) и сохраняет в папку инструментов.
- **`remove_custom_tool`**: Удаляет файл и вычищает его из кэша импортов Python (`sys.modules`).

### Интеграция в мозг (Строки 195–215)
- **`get_custom_tools_prompt`**: Формирует специальный текстовый блок для системного промпта агента. В нем перечислены все установленные вами инструменты и их описания. Так агент узнает, что у него появились "новые способности".

## Как создать свой инструмент?
Создайте файл в папке `~/.tca_custom_tools/` (например, `my_tool.py`):

```python
from langchain_core.tools import tool

@tool
def my_hello_tool(name: str) -> str:
    """Приветствует пользователя по имени. Используй это, когда хочешь быть вежливым."""
    return f"Привет, {name}! Я теперь умею это делать."
```

## Особенности
1. **Динамичность**: Можно вызывать `reload_custom_tools()`, чтобы агент подхватил изменения в ваших скриптах без перезагрузки всего TCA.
2. **Изоляция**: Если ваш кастомный инструмент упадет с ошибкой, загрузчик просто выведет предупреждение в консоль, не ломая основную программу.
