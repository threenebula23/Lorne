# Документация инструмента: code_interpreter.py (Выполнение Python-кода)

> **Для вызова у модели:** тул **`code_interpreter`** не схлопнут в мульти-тул; имя в API совпадает с модулем. Общая карта: [COMPACT_TOOLS.md](../COMPACT_TOOLS.md).

Этот инструмент превращает агента в полноценную среду разработки, позволяя ему запускать Python-код "на лету" для проведения вычислений или проверки идей.

## Для всех: Что это делает?
Это "калькулятор на стероидах". Агент может написать небольшую программу на Python, запустить её и мгновенно увидеть результат.
- Использование: расчеты, обработка данных, проверка сложных алгоритмов.
- Агент видит всё, что код вывел на экран (print), и все ошибки, если они возникли.

---

## Для разработчиков: Разбор кода (подробно)

### Логика работы (Строки 14–42)
Инструмент работает по принципу "создал → запустил → удалил".
1. **Создание (Строки 19–21)**: Текст кода записывается во временный файл (`tempfile.NamedTemporaryFile`) с расширением `.py`.
2. **Запуск (Строки 24–29)**: Используется модуль `subprocess` для запуска этого файла текущим интерпретатором Python (`sys.executable`). Это гарантирует, что у кода будут те же библиотеки, что и у самого TCA.
3. **Захват вывода**: Параметр `capture_output=True` перехватывает всё, что код пишет в консоль (`stdout`) и в поток ошибок (`stderr`).
4. **Таймаут (Строки 35–36)**: Если код завис (например, бесконечный цикл `while True`), инструмент принудительно прервет его через 30 секунд.
5. **Очистка (Строки 39–42)**: Блок `finally` гарантирует, что временный файл будет удален с диска в любом случае.

---

## Особенности
1. **Изоляция**: Код запускается в отдельном процессе. Если в коде произойдет критическая ошибка (например, `exit()`), это не уронит само приложение TCA.
2. **Библиотеки**: Доступны все стандартные библиотеки Python и те, что установлены в окружении TCA.
3. **Безопасность**: Инструмент предназначен для вычислений. Если агент захочет изменить файлы через импорт `os`, он скорее всего воспользуется инструментом `file_ops`, но технически `code_interpreter` тоже имеет доступ к системе в рамках прав пользователя.

## Примеры использования

**Сложный расчет:**
```python
code_interpreter(code="import math; print(math.factorial(100))")
```

**Тестирование регулярного выражения:**
```python
code_interpreter(code="import re; print(re.match(r'\\d+', '123 abc').group())")
```
