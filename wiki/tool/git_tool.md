# Документация инструмента: git_tool.py (Интеграция с Git)

> **Для вызова у модели:** один тул **`git_ops`** (`action`: `status` \| `log` \| `diff` \| `rollback_file`). См. [COMPACT_TOOLS.md](../COMPACT_TOOLS.md). Ниже — отдельные `@tool` в `git_tool.py`, на которые делегирует `git_ops`.

Этот инструмент дает агенту доступ к системе контроля версий Git, позволяя ему видеть историю изменений проекта и отменять неудачные правки.

## Для всех: Что это делает?
Это "машина времени" для кода. Агент может:
- **`git_status`**: Посмотреть, какие файлы изменены и в какой ветке он находится.
- **`git_log`**: Прочитать историю коммитов (кто, когда и зачем менял код).
- **`git_diff`**: Посмотреть разницу между версиями (что именно удалили или добавили).
- **`git_rollback_file`**: Вернуть файл к состоянию, которое было несколько коммитов назад, если текущие правки привели к ошибке.

## Для разработчиков: Разбор кода (подробно)

### Связка с ядром (Строки 7–9)
- Инструмент не работает напрямую с `GitPython`. Вместо этого он вызывает `Agent.git_integration.get_git_manager()`. Это сделано для того, чтобы весь проект использовал одну и ту же логику работы с репозиторием.

### Инструменты

#### `git_log` (Строки 13–19)
- Возвращает список последних коммитов. 
- Параметр `path` позволяет смотреть историю только для конкретного файла.
- Код проверяет `gm.available`. Если папка не инициализирована как Git-репозиторий, инструмент вернет понятную ошибку вместо падения.

#### `git_diff` (Строки 23–29)
- Показывает изменения в классическом формате `diff`.
- Если не передать хеш коммита, покажет текущие незакоммиченные изменения.
- Вывод ограничивается 5000 символами, чтобы не перегружать контекст LLM (длинные диффы обрезаются).

#### `git_rollback_file` (Строки 33–36)
- Выполняет команду отката для конкретного пути. Это безопаснее, чем откат всего проекта целиком.

#### `git_status` (Строки 40–43)
- Возвращает краткую сводку: название текущей ветки, список измененных ("dirty") файлов и список файлов в индексе (staged).

---

## Особенности
1. **Интеграция с TUI**: Статус Git отображается в нижней строке интерфейса TCA (название ветки).
2. **Автоматизация**: В других инструментах (например, `write_file`) есть скрытые вызовы Git для автоматического создания сохранений после каждой записи.

---

## Примеры использования

**Проверить, что изменилось:**
```python
git_status()
```

**Посмотреть историю правок файла:**
```python
git_log(path="main.py", limit=5)
```
