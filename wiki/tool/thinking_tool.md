# Документация инструмента: thinking_tool.py (Размышления и Анализ)

> **Для вызова у модели:** в реестре — **`reasoning_tool`** (`think` \| `diff` \| `analyze`), а не отдельные `think` / `show_diff` / `analyze_code`. См. [COMPACT_TOOLS.md](../COMPACT_TOOLS.md). Ниже — реализация в `thinking_tool.py`.

Этот инструмент помогает агенту "рассуждать вслух", визуализировать планируемые изменения и глубоко анализировать код перед тем, как его менять.

## Для всех: Что это делает?
Это внутренние когнитивные процессы агента, вынесенные наружу.
- **`think`**: Записать мысль или план. Вы увидите это в интерфейсе (панель "Thoughts"). Это помогает агенту не терять нить задачи.
- **`show_diff`**: Сравнить старый код и новый. Прежде чем внести правку в файл "навсегда", агент может показать вам, что именно он собирается изменить (добавленные и удаленные строки).
- **`analyze_code`**: Глубокий поиск по коду. Агент использует RAG (базу знаний кода), чтобы найти все места, где используется определенная функция или переменная.

## Для разработчиков: Разбор кода (подробно)

### `think` (Строки 14–31)
- Самый простой инструмент. Он берет строку текста и передает её в `TUIBridge.on_thought`.
- **Зачем это нужно?** ЛЛМ лучше справляется с задачами, если сначала описывает свои шаги (Chain of Thought). Этот инструмент визуализирует этот процесс для пользователя.

### `show_diff` (Строки 34–66)
- Использует стандартную библиотеку Python `difflib`.
- Генерирует "unified diff" (единый формат различий).
- Вызывает `bridge.on_diff`, что заставляет интерфейс TCA показать красивое окно сравнения файлов.
- Возвращает статистику: сколько строк добавлено (`additions`), сколько удалено (`deletions`).

### `analyze_code` (Строки 69–123)
- Это интерфейс к системе RAG (Retrieval Augmented Generation).
- **Логика**:
  1. Принимает путь к файлу и запрос.
  2. Обращается к `Agent.rag` для поиска наиболее похожих фрагментов кода по всему проекту.
  3. Фильтрует результаты, отдавая приоритет указанному файлу.
  4. Показывает найденные куски кода пользователю через `bridge.on_code`.
- Позволяет агенту "понимать" контекст больших проектов, не читая каждый файл целиком.

## Особенности
1. **Интеграция с UI**: Эти инструменты тесно связаны с `TUIBridge`. Если TCA запущен в режиме IDE, вы увидите визуальные эффекты.
2. **Безопасность**: Эти действия ничего не меняют на диске (read-only), поэтому они не требуют подтверждения пользователя.

---

## Примеры использования

**Запись плана:**
```python
think("Я проанализировал структуру БД. Теперь мне нужно добавить поле 'email' в модель User.")
```

**Проверка изменений:**
```python
show_diff(
    path="models.py",
    old_content="class User:\n    id: int",
    new_content="class User:\n    id: int\n    email: str"
)
```
