# Документация инструмента: web_tool.py (Поиск в интернете)

Этот инструмент позволяет агенту выходить за пределы вашего локального компьютера, чтобы искать информацию в интернете или читать содержимое веб-страниц.

## Для всех: Что это делает?
Это "глаза" агента в глобальной сети.
- **`web_search`**: Как поиск в Google или DuckDuckGo. Агент вводит запрос и получает список сайтов с кратким описанием.
- **`web_fetch`**: Агент переходит по ссылке и "читает" текст на странице, убирая лишнюю рекламу и меню.
- **`web_search_and_read`**: Комбинированный режим — найти и сразу прочитать топовые результаты.

Это полезно, когда нужно найти документацию к новой библиотеке, примеры кода или решение ошибки с StackOverflow.

## Для разработчиков: Разбор кода (подробно)

### Кэширование (Строки 22–41)
Чтобы не нагружать внешние сервисы и работать быстрее, инструмент хранит результаты поиска (`_search_cache`) и загрузки страниц (`_fetch_cache`) в памяти. Время жизни поиска — 5 минут, страницы — 10 минут.

### Очистка HTML (Строки 46–77)
Когда агент "читает" сайт, ему не нужны скрипты, стили или подвалы сайта.
- **`_extract_code_blocks`**: Находит теги `<code>` и `<pre>` и вытаскивает оттуда чистый программный код.
- **`_html_to_text`**: Использует регулярные выражения для удаления мусора (реклама, навигация, хедеры) и конвертирует HTML в читаемый текст.

### Инструменты LangChain

#### `web_search` (Строки 83–124)
Использует библиотеку `ddgs` (DuckDuckGo Search).
- **Умный поиск (Строки 97–102)**: Если в запросе есть слова "docs", "api" или "tutorial", инструмент автоматически добавляет к запросу фильтр по сайтам (например, `site:docs.python.org` или `site:github.com`), чтобы найти более качественную информацию.

#### `web_fetch` (Строки 130–172)
Загружает содержимое конкретного URL.
- **Безопасность**: Использует стандартный `urllib.request` с поддельным заголовком `User-Agent`, чтобы сайты не блокировали "бота".
- **Ограничения**: Если текст слишком длинный (больше 8000 символов), он обрезается, чтобы не переполнять память LLM.

#### `web_search_and_read` (Строки 178–228)
Самый мощный инструмент. Он делает поиск, берет первые несколько ссылок и по очереди загружает их содержимое в один большой ответ. Это экономит время агента на лишние размышления и запросы.

## Особенности
- **DuckDuckGo**: Поиск работает через DuckDuckGo, так как он не требует API-ключей и достаточно быстрый.
- **Чистка текста**: Инструмент старается максимально эффективно сжимать информацию, чтобы передавать агенту только суть.

## Примеры использования

**Поиск документации:**
```python
web_search(query="langchain tool definition example")
```

**Чтение конкретной статьи:**
```python
web_fetch(url="https://docs.python.org/3/library/pathlib.html")
```
