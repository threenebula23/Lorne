# Документация инструмента: terminal_tool.py (Командная строка)

> **Для вызова у модели:** **`run_command`** — отдельный тул, не часть мульти-диспетчеров. См. [COMPACT_TOOLS.md](../COMPACT_TOOLS.md).

Этот инструмент дает агенту возможность выполнять команды в терминале вашей операционной системы (cmd в Windows или bash/sh в macOS и Linux).

## Для всех: Что это делает?
Это "запрос на действие" в компьютерной системе. С его помощью агент может:
- Запускать ваши программы (например, `python main.py`).
- Устанавливать новые библиотеки (`pip install requests`).
- Проверять установленные программы (`git --version`).
- Выполнять любые другие команды, которые вы обычно вводите в черное окно терминала.

**ВАЖНО:** Агент **всегда** спрашивает ваше разрешение перед запуском любой команды. Без вашего "да" в интерфейсе ничего не выполнится.

## Для разработчиков: Разбор кода (подробно)

### Защита и безопасность
- **Дедупликация (опционально):** переменная окружения **`TCA_RUN_COMMAND_DEDUPE_S`**. Если задано **положительное** число секунд, повтор **той же** сигнатуры команды внутри этого окна будет отклонён (`_too_soon`). Значение **`0`** (по умолчанию) — дедуп **выключен**, повторы не блокируются. См. `Agent/tools/terminal_tool.py`.
- **`_RECENT`**: словарь для отслеживания недавних команд при включённом дедупе.
- **`_is_dangerous` (Строки 47–59)**: Список "запрещенных" команд. Если команда содержит `rm -rf /`, `mkfs` или другие разрушительные паттерны, она блокируется автоматически даже без запроса подтверждения.

### Основная функция `run_command` (Строки 62–135)
- **Входные данные**: `command` (сама команда), `cwd` (папка, в которой её запустить), `timeout_seconds` (максимальное время ожидания).
- **Логика подтверждения (Строки 94–124)**:
  - Инструмент проверяет переменную `AUTO_CONFIRM`. Если она `False`, он обращается к `TUIBridge` (интерфейсу), чтобы показать пользователю окно подтверждения.
  - Если запуск идет в консольном режиме (без графики), он просто просит ввод `y/n` в терминале.
- **Выполнение (Строка 133)**: Если подтверждено, вызывается `run_command_safe` из модуля `Terminal.runner`. Этот модуль отвечает за кроссплатформенность и безопасный захват вывода (stdout/stderr).
- **Результат**: Возвращается словарь, содержащий:
  - `stdout`: Весь текст, который программа вывела в консоль.
  - `stderr`: Текст ошибок.
  - `returncode`: Числовой код завершения (0 — успех, остальное — ошибка).

## Особенности
1. **Интерактивность**: Если приложение запущено не в реальном терминале (например, в какой-то среде без ввода), инструмент вернет ошибку `EOF`, так как он не может спросить подтверждение.
2. **Рабочая директория (CWD)**: Агент может переходить в подпапки проекта для сборки или тестирования, передавая параметр `cwd`.

---

## Примеры использования

**Запуск тестов:**
```python
run_command(command="pytest tests/", timeout_seconds=60)
```

**Проверка зависимостей:**
```python
run_command(command="pip freeze")
```
